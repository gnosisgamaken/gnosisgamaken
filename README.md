# Pedro Costa — operator-led software, AI systems, and real-world operations

I build software from inside the mess of real businesses.

Most of my work starts with an operational problem: bookings that do not reconcile, customers that fall through the cracks, tours that need better delivery, teams that need clearer signals, or old workflows that deserve to become systems. I like tools that make a business more legible.

## Current focus

- PlayGranada operating systems: booking flows, customer intelligence, tour operations, and conversion infrastructure.
- Audio Tickets: multilingual audio-guide/ticket delivery, content QA, and customer-facing tour experiences.
- Brain / CRM: turning fragmented customer and operations data into useful memory.
- AI-assisted operations: using agents as a practical control room for audits, QA, monitoring, documentation, and workflow automation.
- Rabbit R1 / native UX experiments: small-device interaction patterns, capture flows, and role-based interfaces.

## The through-line

A lot of my projects are attempts to answer the same question:

> How do we turn messy real-world work into systems that people can actually use?

That started early with OSCI, a PHP/MySQL POS and stock-management system for Oteiza Coffee. It was my first managed software-development project and, in hindsight, a seed of everything I still care about: operator-led tools, business-specific workflows, and software that reflects how work really happens.

## How I work

- Start from the operational reality, not the ideal diagram.
- Preserve useful history before cleaning things up.
- Prefer boring, inspectable systems over fragile magic.
- Use AI agents for leverage, but keep human judgment in the loop where stakes are real.
- Treat documentation as part of the system, not an afterthought.
